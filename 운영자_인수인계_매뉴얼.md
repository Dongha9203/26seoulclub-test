# 동아리ON 챗봇 — 운영자 인수인계 매뉴얼

> 이 문서는 챗봇/관리자 대시보드 "사용법"이 아니라, 시스템을 운영자가 직접
> 소유·운영할 수 있도록 넘기는 절차서입니다. (사용법은 대시보드 내 안내문구와
> 별도 사용자 매뉴얼을 참고하세요.)

## 0. 지금 상황 한눈에 보기

| 항목 | 현재 상태 | 인수인계 시 해야 할 일 |
|---|---|---|
| Supabase (DB) | 운영자 계정 소유 | 그대로 사용 — 추가 작업 없음 |
| Anthropic API 키 | 운영자 계정 소유 | 그대로 사용 |
| Voyage AI API 키 | 운영자 계정 소유 | 그대로 사용 |
| Notion 통합(integration) | 운영자 워크스페이스에 생성됨 | 그대로 사용 — 단, **새 페이지를 만들 때마다 공유 필요** (4-a 참고) |
| Voyage AI 결제수단 | **미등록** | 등록 필요 (4-f 참고) — 안 하면 노션 동기화 시 임베딩이 조용히 실패할 수 있음 |
| GitHub 저장소 | 최신 코드는 개발자 개인 저장소에만 있음. `seoulclub-on/26seoulclub`에는 일부만 반영됨 | 개발자가 최신 코드를 `seoulclub-on/26seoulclub`에 푸시 |
| Vercel 배포 | 개발자 개인 Vercel 계정으로 테스트 중 | 운영자 Vercel 계정에 새 프로젝트로 처음부터 연결 |

핵심: DB/API 키/Notion은 이미 운영자 소유라 옮길 게 없습니다. **실제로 옮겨야 하는 건
"코드"와 "그 코드를 띄우는 Vercel 프로젝트" 두 가지뿐**입니다.

## 1. 개발자가 운영자에게 전달해야 할 것

1. **코드**: GitHub 저장소 `seoulclub-on/26seoulclub`의 기본 브랜치에 최종 코드를 푸시.
   (이 저장소는 이미 운영자 소유이므로 따로 "전달"할 필요 없이 푸시만 하면 끝.)

2. **환경변수 5종 값** — 전달 방식 주의:
   - 카카오톡/이메일 등에 평문으로 보내지 말 것. 화면 공유하면서 운영자가 직접
     Vercel 화면에 입력하도록 안내하거나, 비밀번호 관리 도구(1Password 등)로 전달.
   - `NOTION_API_TOKEN`
   - `ANTHROPIC_API_KEY`
   - `VOYAGE_API_KEY`
   - `SUPABASE_DB_URL`
   - `JWT_SECRET_KEY` — **운영 배포용으로 새로 발급**해서 전달 (테스트용 키 재사용 금지).
     발급 명령: `python -c "import secrets; print(secrets.token_urlsafe(48))"`

3. **최초 운영자 로그인 계정**: 개발자가 `create_admin_account.py <이메일> <비밀번호>`를
   미리 실행해서 만들거나, 운영자와 화면을 보며 함께 실행. (가입 화면이 따로 없는
   시스템이라 이 스크립트로만 최초 계정이 만들어집니다.)

4. 이 문서.

---

## 2. 운영자가 직접 해야 할 일 — Vercel에 배포하기 (최초 1회)

1. [vercel.com](https://vercel.com) 에 본인(운영자) 계정으로 로그인
2. **Add New...** → **Project**
3. GitHub 저장소 `seoulclub-on/26seoulclub` 선택해서 Import
   - 목록에 안 보이면: GitHub → **Settings** → **Applications** → **Vercel** → **Configure**에서
     Repository access에 `26seoulclub`이 추가되어 있는지 확인 (없으면 추가 후 Save)
4. Framework Preset: 자동 감지 안 되면 **Other** 선택
5. **Environment Variables**에 1번에서 전달받은 5개 값을 모두 입력
   (Production / Preview / Development 전부 체크)
6. **Deploy** 클릭
7. 배포 완료 후 나온 주소(예: `https://26seoulclub.vercel.app`) 확인

---

## 3. 배포 후 1회만 하면 되는 작업

1. **챗봇 동작 확인**: `{배포주소}/widget.html` 접속해서 질문 1개 보내보기
2. **관리자 로그인 확인**: `{배포주소}/admin/login.html`에서 1-3번 계정으로 로그인
3. 둘 중 하나라도 안 되면 → Vercel 대시보드의 **Deployments → 실패 로그**를
   캡처해서 개발자에게 전달
4. 로그인 성공하면 대시보드 내 **비밀번호 변경** 메뉴에서 전달받은 비밀번호를
   본인만 아는 비밀번호로 즉시 변경
5. **노션 임베드 주소 교체**: 기존에 노션 페이지에 넣어둔 위젯 임베드 블록의 URL을
   `{새 배포주소}/widget.html`로 바꿔주기 (배포 계정이 바뀌면 주소도 바뀝니다)
6. (선택) Vercel 프로젝트 설정에서 커스텀 도메인 연결 가능 — 원하면 진행

## 4. 운영 중 알고 있어야 할 것

### a) 노션에 새 정보를 추가할 때
- 이미 연결된 2개 페이지("메인페이지", "통합시스템") **안에** 하위 페이지/토글/섹션을
  추가하는 건 자동으로 챗봇에 반영됩니다(매일 새벽 자동 갱신 또는 대시보드 "지금
  갱신" 버튼).
- 단, 그 페이지가 **다른 사람이 새로 만들었거나 워크스페이스 다른 곳에서 옮겨온
  페이지**라면, 그 페이지를 열어 우측 상단 **"..." → "연결(Connections)"**에
  챗봇 통합이 추가되어 있는지 꼭 확인하세요. 빠지면 챗봇이 그 내용을 영원히
  찾지 못합니다. (실제로 "강사비 지급 기준" 페이지가 이 문제였습니다.)
- 완전히 새로운 **최상위** 페이지(메인페이지/통합시스템과 같은 레벨)를 추가로
  연결하려면 코드 수정이 필요하니 개발자에게 요청하세요.
- **Airtable과 노션 데이터베이스는 서로 다른 기능**이고, 챗봇이 읽는 범위도 다릅니다.
  - 노션 페이지에 **Airtable 표/폼을 임베드**해 넣은 경우 → 표 안 데이터는
    못 읽고, 그 자리에 있던 링크만 "관련 링크: https://..." 형태로 검색에 남습니다.
  - 노션 자체의 **데이터베이스(표 보기) 기능**으로 만든 자료(예: "자료실")는
    각 행(row)을 별도 페이지로 보고 그 안의 내용을 챗봇이 읽어 검색에
    활용합니다. 행 추가/삭제/수정도 매일 새벽 자동 갱신에 반영됩니다.

### b) 자동 갱신은 하루 1번뿐입니다 (매일 새벽 4시, Vercel 무료 플랜 제약)
- 급하게 반영해야 하면 대시보드의 **"지금 갱신"** 버튼을 누르면 즉시 동기화됩니다.

### c) 1년이 지난 질의응답 기록은 자동 삭제됩니다 (개인정보 보호 목적)

### d) 코드 수정 없이 운영자가 바로 바꿀 수 있는 것 (대시보드에서 즉시 반영)
- 담당자 연락처, 신뢰도 판단 기준, 욕설/혐오 표현 사전, 질문 상황 분류 키워드,
  최대 질문 글자수, 분당 요청 제한(같은 사용자가 1분에 몇 번까지 질문 가능한지) 등
  — 대시보드 "API 파라미터" 메뉴에서 저장하면 재배포 없이 바로 다음 질문부터 반영됩니다.

### e) 반드시 개발자에게 요청해야 하는 것 (코드 수정 + 재배포 필요)
- 새로운 최상위 노션 페이지 연결, 새 기능 추가, 디자인 변경 등

### f) Voyage AI 결제수단 등록 (필수 권장)
- 챗봇이 노션 문서를 검색용으로 변환(임베딩)할 때 Voyage AI를 사용합니다.
  결제수단이 등록되지 않은 계정은 **분당 3건, 분당 1만 토큰**으로 제한되는데,
  노션 동기화 시 한 번에 갱신되는 문서가 많으면 이 제한에 걸려 일부 문서가
  **검색에 반영되지 않고 조용히 실패**할 수 있습니다(에러 화면 없이 그냥 안 보임).
- [Voyage AI 대시보드](https://dashboard.voyageai.com/) → 결제(Billing) 페이지에서
  결제수단을 등록하면 제한이 크게 늘어납니다. 무료 토큰(voyage-3 계열 기준 2억
  토큰)은 결제수단 등록 후에도 그대로 적용되니 비용 걱정 없이 등록해도 됩니다.
