## 1. 사전 환경 설정 (최초 1회)

### 사전 조건 — Claude Pro 또는 Max 플랜 구독 필수

Claude Code를 사용하려면 **[claude.ai](https://claude.ai)에서 Pro 또는 Max 플랜 구독**이 되어 있어야 합니다. 구독 없이는 Claude Code를 실행할 수 없습니다. 아래 설치 전에 먼저 구독 여부를 확인하세요.

---

### 1-1. Claude Code 설치 (두 가지 방법 중 하나 선택)

#### 방법 A — 데스크탑 앱 (권장, PowerShell 불필요)

1. [claude.ai/download](https://claude.ai/download) 접속 → **Windows** 버전 다운로드 후 설치
2. 앱 실행 → Claude 계정(Pro/Max)으로 로그인
3. 프로젝트 폴더(`donghoi-on-chatbot`)를 선택하면 바로 사용 가능

> 데스크탑 앱은 PowerShell을 따로 열지 않아도 앱 안에서 Claude Code를 바로 실행할 수 있습니다.

#### 방법 B — 터미널(CLI) 방식

Node.js를 먼저 설치해야 합니다.

1. [nodejs.org](https://nodejs.org) 접속 → **LTS** 버전 다운로드 후 설치
2. 설치 완료 확인: PowerShell을 열고 아래 입력
   ```
   node -v
   ```
   버전 번호(예: `v22.0.0`)가 나오면 설치 완료
3. Claude Code 설치:
   ```
   npm install -g @anthropic/claude-code
   ```
4. 설치 완료 확인:
   ```
   claude -v
   ```
   버전 번호가 나오면 설치 완료
5. Claude Code 로그인:
   ```
   claude
   ```
   실행 후 로그인 화면에서 claude.ai 계정(Pro/Max) 연결

### 1-2. Git 설치 (데스크탑 앱·CLI 모두 필요)

코드를 GitHub에 올릴 때 사용합니다. 데스크탑 앱에서 Claude Code에게 "GitHub에 올려줘"라고 요청할 때도 내부적으로 git이 실행되므로 반드시 설치해야 합니다.

1. [git-scm.com](https://git-scm.com) 접속 → 다운로드 후 설치 (기본 설정 그대로 Next)
2. 설치 완료 확인: PowerShell에서
   ```
   git -v
   ```
   버전 번호가 나오면 설치 완료
3. 최초 1회 이름·이메일 등록 (커밋 기록에 표시됨):
   ```
   git config --global user.name "홍길동"
   git config --global user.email "이메일@example.com"
   ```

4. **GitHub 로그인 연결** (처음 push 전 필수):
   코드를 GitHub에 올리려면 계정 인증이 필요합니다. git-scm.com에서 기본 설정으로 설치하면 **Git Credential Manager**가 자동으로 함께 설치됩니다.
   - 처음으로 `git push`를 실행하면 **브라우저 창이 자동으로 열리면서** GitHub 로그인 화면이 나타납니다.
   - GitHub 계정으로 로그인 후 **"Authorize git-credential-manager"** 버튼을 클릭하면 인증 완료입니다. 이후에는 자동으로 처리됩니다.

   > **⚠️ 브라우저 창이 열리지 않거나 인증 오류가 날 경우** — GitHub에서 직접 토큰을 발급해야 합니다.
   > 1. GitHub.com 로그인 → 우측 상단 프로필 사진 클릭 → **Settings**
   > 2. 좌측 하단 **Developer settings** → **Personal access tokens** → **Tokens (classic)** → **Generate new token (classic)**
   > 3. Note 칸에 이름 입력(예: "챗봇유지보수"), 만료기간 선택, **repo** 체크박스에 체크 → **Generate token** 클릭
   > 4. 화면에 표시된 토큰을 복사해 안전한 곳에 보관 (이후 재확인 불가)
   > 5. 이후 `git push` 실행 시 username에 GitHub 아이디, password 자리에 복사한 토큰을 입력

### 1-4. Supabase MCP 연동 (선택 — 설정해두면 DB 조회가 훨씬 편해집니다)

#### 무엇인가요?

MCP(Model Context Protocol)는 Claude Code가 외부 서비스에 직접 접속할 수 있게 해주는 연결 방식입니다. Supabase MCP를 연동해두면 Claude Code가 챗봇 DB에 직접 접속해서 데이터를 조회하고 분석할 수 있습니다.

**설정 전**: Claude Code에게 DB를 물어보면 코드를 작성해서 실행하는 방식으로 간접 조회
**설정 후**: Claude Code가 DB에 직접 접속해서 바로 결과를 가져옴 — 더 빠르고 정확합니다

**유용한 상황 예시:**
- "이번 주 질의응답이 몇 건이야?" → 바로 숫자를 알려줌
- "어떤 질문이 가장 많이 실패했어?" → qa_log를 직접 분석해서 정리해줌
- "DB 용량 현황 확인해줘" → 테이블별 용량을 즉시 알려줌
- "지식DB에 어떤 문서가 등록되어 있어?" → documents 테이블을 바로 조회

---

#### 설정 방법 (최초 1회, 약 10분)

**① Supabase Personal Access Token 발급**

1. [supabase.com](https://supabase.com) 로그인
2. 우측 상단 계정 아이콘 클릭 → **"Account"**
3. 좌측 메뉴 **"Access Tokens"** 클릭
4. **"Generate new token"** 클릭
5. 이름 입력 (예: `claude-code-mcp`) → **"Generate token"**
6. 화면에 표시된 토큰 값을 복사해서 안전한 곳에 저장
   > ⚠️ 이 화면을 닫으면 토큰 값을 다시 볼 수 없습니다. 반드시 복사해두세요.
   > 외부에 유출되면 Supabase 프로젝트 전체에 접근 권한이 생기므로 카카오톡·이메일에 평문으로 저장하지 마세요.

---

**② Claude Code에 Supabase MCP 추가**

**방법 A — 데스크탑 앱을 쓰는 경우**

1. Claude Code 데스크탑 앱 실행
2. 좌측 하단 설정 아이콘(⚙️) 클릭 → **"Developer"** 탭 → **"Edit Config"** 버튼 클릭
3. 메모장 등의 편집기로 파일이 열립니다. 파일 안에 아래 내용을 추가합니다:

   파일이 `{}` 처럼 비어있는 경우:
   ```json
   {
     "mcpServers": {
       "supabase": {
         "command": "npx",
         "args": [
           "-y",
           "@supabase/mcp-server-supabase@latest",
           "--access-token",
           "여기에_①에서_복사한_토큰_붙여넣기"
         ]
       }
     }
   }
   ```

   파일에 이미 다른 내용이 있는 경우 — `"mcpServers"` 항목만 추가합니다:
   ```json
   {
     "기존내용": "...",
     "mcpServers": {
       "supabase": {
         "command": "npx",
         "args": [
           "-y",
           "@supabase/mcp-server-supabase@latest",
           "--access-token",
           "여기에_①에서_복사한_토큰_붙여넣기"
         ]
       }
     }
   }
   ```

4. 파일 저장 후 **Claude Code 데스크탑 앱을 완전히 종료 후 재시작**

---

**방법 B — 터미널(CLI) 방식을 쓰는 경우**

PowerShell을 열고 아래 명령을 실행합니다 (토큰 부분은 ①에서 복사한 값으로 교체):

```
claude mcp add supabase -- npx -y @supabase/mcp-server-supabase@latest --access-token 여기에_토큰_붙여넣기
```

실행 후 `claude`를 다시 시작하면 적용됩니다.

---

**③ 연동 확인**

Claude Code를 시작한 뒤 아래와 같이 물어보세요:

```
> Supabase DB에 접속해서 qa_log 테이블에서 가장 최근 질의 5건을 보여줘
```

데이터가 바로 출력되면 연동 성공입니다.

오류가 나는 경우:
- `npx` 명령을 찾을 수 없다는 오류 → Node.js가 설치되어 있지 않습니다. **1-1 방법 B**의 Node.js 설치 단계를 먼저 진행하세요.
- 인증 오류 → 토큰 값을 다시 확인하세요. 토큰 앞뒤 공백이 없어야 합니다.

---

#### 연동 후 Claude Code에게 요청하는 예시

```
> 이번 달 질의응답 건수와 성공률을 알려줘
```
```
> qa_log에서 failure_cause별로 건수를 집계해줘 — 어떤 유형이 가장 많아?
```
```
> documents 테이블에 등록된 문서 목록을 source_type별로 보여줘
```
```
> DB 테이블별 용량 현황을 확인해줘
```
```
> 오늘 23시 이후 들어온 질의 중 latency_ms가 10초 이상인 건 있어?
```

> **참고**: Supabase MCP는 읽기(SELECT)와 쓰기(INSERT/UPDATE/DELETE) 모두 가능합니다. 데이터를 삭제하거나 수정하는 작업은 Claude Code가 실행 전 반드시 내용을 먼저 설명하므로, 확인 후 승인 또는 거부할 수 있습니다.

## 2. 프로젝트 파일 구조

코드 수정을 요청하기 전, 어떤 파일이 어떤 역할인지 파악해두면 Claude Code에게 더 정확하게 요청할 수 있습니다.

```
donghoi-on-chatbot/
│
├── config.json              ← 챗봇 설정 (노션 URL, 구글 캘린더, AI 모델 등)
├── forbidden_words.json     ← 초기 시드 파일 (운영 중 관리는 대시보드 → 금지어 사전)
├── situation_keywords.json  ← 초기 시드 파일 (운영 중 관리는 대시보드 → 분류 키워드)
├── vercel.json              ← Vercel 배포 설정
├── requirements.txt         ← Python 패키지 목록
├── .env.example             ← 환경변수 목록 예시 (실제 값은 Vercel 대시보드에서 관리)
│
├── api/                     ← API 서버 (Vercel이 실행하는 핵심 코드)
│   ├── chat.py              ← 챗봇 대화 처리
│   ├── admin.py             ← 관리자 대시보드 API
│   ├── sync_notion.py       ← 수동 노션 동기화 ("지금 갱신" 버튼)
│   └── cron/
│       └── sync_notion.py   ← 자동 동기화 (매일 새벽 4시 크론)
│
├── collectors/              ← 데이터 수집기
│   ├── notion_collector.py  ← 노션 페이지 수집
│   ├── calendar_collector.py← 구글 캘린더 수집
│   ├── google_sheet_collector.py ← 구글 스프레드시트 수집
│   ├── pdf_collector.py     ← PDF 파일 수집
│   ├── docx_collector.py    ← Word 파일 수집
│   ├── excel_collector.py   ← Excel 파일 수집
│   └── airtable_collector.py← Airtable 수집 (현재 비활성)
│
├── storage/                 ← Supabase DB 연결 및 쿼리
├── public/                  ← 화면 파일
│   ├── widget.html          ← 챗봇 위젯 (노션에 임베드되는 화면)
│   └── admin/              ← 관리자 대시보드 화면
│       ├── index.html
│       ├── login.html
│       ├── app.js
│       └── styles.css
│
├── chatbot_engine.py        ← 챗봇 핵심 로직 (검색 → 프롬프트 → Claude API 호출)
├── auth.py                  ← 관리자 인증
├── build_embeddings.py      ← 문서 임베딩(벡터화) 처리
├── hybrid_search.py         ← 검색 엔진 (벡터+키워드 혼합)
├── prompt_builder.py        ← Claude API에 보내는 프롬프트 구성
├── failure_analyzer.py      ← 실패 원인 분류 로직
└── models/, utils/          ← 내부 공통 코드
```

### 파일 수정 기준

| 구분 | 파일 | 방법 |
|------|------|------|
| **직접 수정 가능** | `config.json` | GitHub 웹 편집기 또는 텍스트 편집기로 직접 수정 |
| **Claude Code로 수정** | `api/`, `collectors/`, `public/`, `*.py` 전체 | 터미널에서 Claude Code 실행 후 요청 |
| **수정하지 않음** | `forbidden_words.json`, `situation_keywords.json`, `vercel.json`, `requirements.txt`, `.env.example` | 앞 두 파일은 최초 DB 시드용이라 수정해도 런타임에 반영되지 않음(대시보드에서 관리). 나머지는 잘못 수정 시 배포가 깨질 수 있음 |

## 3. Claude Code 시작하기

### 방법 A — 데스크탑 앱으로 시작 (PowerShell 불필요)

1. **Claude Code 데스크탑 앱** 실행
2. 좌측 상단 폴더 아이콘 또는 **"Open Folder"** 클릭
3. `donghoi-on-chatbot` 폴더 선택
4. 채팅창이 열리면 한국어로 바로 요청하면 됩니다:
   ```
   챗봇 답변이 너무 길어. 최대 300자로 줄여줘
   ```
5. **종료**: 앱 창 닫기

---

### 방법 B — 터미널(PowerShell)로 시작

1. **Windows 터미널(PowerShell)** 또는 **명령 프롬프트** 열기
   - 시작 메뉴에서 "PowerShell" 검색 후 실행

2. **프로젝트 폴더로 이동**
   ```
   cd "C:\Users\[사용자명]\donghoi-on-chatbot"
   ```
   (폴더 경로는 실제 저장 위치에 맞게 수정)

3. **Claude Code 시작**
   ```
   claude
   ```

4. **대화 시작** — 프롬프트(`>`)가 나타나면 한국어로 요청하면 됩니다.
   ```
   > 챗봇 답변이 너무 길어. 최대 300자로 줄여줘
   ```

5. **종료**: `/exit` 입력 또는 `Ctrl + C`

### 유용한 명령어 (방법 B — 터미널 방식)

| 명령어 | 설명 |
|--------|------|
| `claude` | Claude Code 시작 |
| `/help` | 도움말 |
| `/clear` | 대화 내용 초기화 (새 주제 시작 시) |
| `/exit` | 종료 |

## 4. Claude Code로 작업하기

### 4-1. 요청하는 방법

Claude Code는 프로젝트 전체 코드를 읽고 문맥을 파악합니다. 파일 이름을 몰라도 됩니다. 어떤 상황인지, 어떻게 바뀌면 좋겠는지를 자연어로 설명하면 됩니다.

**기본 요청 형식:**
```
현재 [어떤 상황]인데, [어떻게 바꾸고 싶다].
```

- **한 번에 하나씩**: 여러 수정을 동시에 요청하면 결과 확인이 어렵습니다. 하나 수정 → 배포 확인 → 다음 수정 순서로 진행하세요.

## 5. 수정 후 배포하기

코드를 수정했으면 GitHub에 올려야 Vercel이 자동으로 새 버전을 배포합니다.

### 5-1. Claude Code 안에서 바로 커밋·푸시

Claude Code가 파일을 수정한 직후, 아래처럼 요청하면 됩니다:
```
> 방금 수정한 내용을 GitHub에 올려줘. 커밋 메시지는 "챗봇 답변 길이 제한 수정"으로.
```
Claude Code가 `git add` → `git commit` → `git push`까지 자동으로 처리합니다.

### 5-2. 터미널에서 직접 하는 방법

직접 하고 싶을 때는 아래 명령을 순서대로 입력합니다:
```
git add .
git commit -m "수정 내용 한 줄 요약"
git push
```

### 5-3. 배포 확인

1. [vercel.com](https://vercel.com) → 해당 프로젝트 → **Deployments** 탭
2. 목록 맨 위 항목이 **"Ready"**(초록 체크)이면 배포 성공
3. **"Error"**(빨간 ×)이면 해당 항목 클릭 → 빌드 로그 확인
   - 오류 메시지를 복사해서 Claude Code에게 붙여넣고 "이 오류 고쳐줘" 요청

### 5-4. 배포 완료 후 동작 확인

**배포 주소 확인 방법:** [vercel.com](https://vercel.com) → 해당 프로젝트 → 상단 **Domains** 항목에 표시된 주소가 운영 URL입니다.  
(예: `https://26seoulclub.vercel.app` 형태 또는 설정된 커스텀 도메인)

해당 주소 뒤에 `/widget.html`을 붙인 URL(`{배포주소}/widget.html`)에서 직접 질문 1개 보내서 정상 응답 확인

### c) Supabase 프로젝트 일시정지 복구

DB 용량(500 MB)이 초과되면 Supabase가 프로젝트를 자동 일시정지합니다. 이 경우 챗봇과 대시보드가 모두 중단됩니다.

**복구 절차:**

1. [supabase.com](https://supabase.com) 로그인 → 해당 프로젝트 선택
2. 화면에 **"Project paused"** 또는 **"Restore project"** 버튼이 표시됨
3. **"Restore project"** 버튼 클릭
4. 복구 완료까지 **5~10분** 대기 (자동으로 진행)
5. 완료 후 챗봇 위젯에서 질문 1개로 정상 동작 확인

**복구 후 반드시 할 것: 용량 확보**
복구해도 용량이 여전히 초과 상태면 곧 다시 일시정지됩니다. 복구 직후 아래 **6-d 항목**을 따라 용량을 정리하세요.

### d) DB 용량 임박 시 정리

Supabase 무료 플랜 한도는 **500 MB**입니다.
사용량이 75%(375 MB), 90%(450 MB)에 도달하면 프로젝트 소유자 이메일로 자동 경고 메일이 발송됩니다.
75% 경고 메일을 받으면 정리를 시작하는 것이 좋습니다.

**현재 용량 확인:**
[supabase.com](https://supabase.com) → 프로젝트 → **Settings → Usage**

**정리 방법 ① — 어드민 대시보드에서 삭제 (권장)**
- 대시보드 → **KB 데이터목록** → 더 이상 사용하지 않는 파일·구글시트 문서 삭제
- 노션 소스 문서는 노션에서 직접 페이지를 삭제한 뒤 "지금 갱신"을 눌러 DB에서도 제거

**정리 방법 ② — Supabase SQL Editor에서 직접 정리**

대시보드로 삭제할 수 없는 항목(크론 기록 등)은 Supabase SQL Editor를 사용합니다:

1. [supabase.com](https://supabase.com) → 프로젝트 → 좌측 메뉴 **SQL Editor**
2. 아래 쿼리 중 필요한 것을 붙여넣고 **Run** 클릭

```sql
-- 크론 실행 기록 중 90일 초과 항목 삭제 (안전)
DELETE FROM cron_run_log
WHERE created_at < NOW() - INTERVAL '90 days';
```

```sql
-- 현재 테이블별 용량 확인 (삭제 전 현황 파악용)
SELECT
  relname AS table_name,
  pg_size_pretty(pg_total_relation_size(relid)) AS total_size
FROM pg_catalog.pg_statio_user_tables
ORDER BY pg_total_relation_size(relid) DESC;
```

> **주의**: SQL Editor에서 실행한 쿼리는 즉시 반영되며 되돌릴 수 없습니다. 확신이 없으면 Claude Code에게 "이 쿼리가 안전한지 확인해줘" 라고 먼저 물어보세요.

**정리 방법 ③ — Claude Code에게 진단 요청**

어떤 데이터가 얼마나 차지하는지 파악하고 싶을 때:
```
> Supabase DB 용량이 많이 찼어. 어떤 테이블이 가장 많이 차지하는지 확인하는 SQL을 알려주고, 안전하게 정리할 수 있는 방법을 알려줘.
```

## 7. 자주 받는 질문

**Q. Claude Code가 코드를 잘못 수정했을 때 어떻게 되돌리나요?**

A. Claude Code에게 직접 요청합니다:
```
> 방금 수정한 내용이 마음에 안 들어. 원래대로 되돌려줘.
```
또는 GitHub에서 이전 커밋으로 되돌릴 수 있습니다:
```
git revert HEAD
git push
```
Claude Code에게 "직전 커밋으로 되돌려줘"라고 요청해도 됩니다.

**Q. 수정 내용을 배포하기 전에 미리 테스트할 수 있나요?**

A. `master` 브랜치에 푸시하면 Vercel이 즉시 운영 서버에 반영합니다. 따라서 **코드를 먼저 테스트하려면 별도의 브랜치를 만들어야 합니다.**

미리 테스트하고 싶을 때 Claude Code에게 아래와 같이 요청하세요:
```
> 수정 사항을 바로 운영에 올리기 전에 먼저 테스트해보고 싶어. 새 브랜치를 만들어서 GitHub에 올려줘.
```
Claude Code가 새 브랜치를 만들어 푸시하면 Vercel이 별도의 임시 URL(프리뷰)로 배포합니다. 확인 방법:
1. Vercel 대시보드 → **Deployments** → 목록 맨 위 항목 옆 **Visit** 버튼 클릭 → 임시 URL에서 동작 확인
2. 이상 없으면 Claude Code에게 요청:
   ```
   > 테스트 브랜치 내용을 운영 버전(master 브랜치)에 반영해줘.
   ```

**Q. config.json을 수정했는데 적용이 안 돼요.**

A. `config.json`은 Vercel 재배포가 있어야 반영됩니다. GitHub에서 파일을 수정·커밋하면 자동으로 재배포됩니다. (대시보드의 "지금 갱신"만으로는 `config.json` 변경이 반영되지 않습니다.)

**Q. Anthropic API 크레딧은 어디서 충전하나요?**

A. [console.anthropic.com](https://console.anthropic.com) → **Billing** → **Add credits**에서 충전합니다.

**Q. Voyage AI 크레딧은 어디서 충전하나요?**

A. [dashboard.voyageai.com](https://dashboard.voyageai.com) → **Billing** 탭에서 충전합니다. Voyage AI는 챗봇이 질문을 이해하는 데 쓰이는 임베딩 서비스로, Anthropic 크레딧과 별도로 관리됩니다. 크레딧이 소진되면 챗봇이 질문에 응답하지 않으니 주기적으로 잔량을 확인하세요.
