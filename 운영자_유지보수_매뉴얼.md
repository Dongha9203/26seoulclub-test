# 동아리ON 챗봇 — 운영자 유지보수 매뉴얼

> 이 문서는 챗봇 시스템을 스스로 수정·보완·복구하는 방법을 안내합니다.
> 코드 수정은 AI 도구(Claude Code)를 활용하며, 별도 개발자 없이 운영자가 직접 처리합니다.

| 상황 | 해당 섹션 |

## 1. 사전 환경 설정 (최초 1회)

코드 수정을 위해 아래 프로그램을 PC에 한 번만 설치하면 됩니다.

### 1-1. Node.js 설치
1. [nodejs.org](https://nodejs.org) 접속 → **LTS** 버전 다운로드 후 설치
2. 설치 완료 확인: Windows 터미널(PowerShell)을 열고 아래 명령 입력
   ```
   node -v
   ```
   버전 번호(예: `v22.0.0`)가 나오면 설치 완료

### 1-2. Git 설치
1. [git-scm.com](https://git-scm.com) 접속 → 다운로드 후 설치 (기본 설정 그대로 Next)
2. 설치 완료 확인:
   ```
   git -v
   ```
   버전 번호가 나오면 설치 완료
3. 최초 1회 이름·이메일 등록 (커밋 기록에 표시됨):
   ```
   git config --global user.name "홍길동"
   git config --global user.email "이메일@example.com"
   ```

### 1-3. 소스코드 폴더 위치 확인
전달받은 `donghoi-on-chatbot` 폴더를 PC에 보관합니다.
이 폴더가 GitHub 저장소(`seoulclub-on/26seoulclub`)와 연결되어 있어야 합니다.

연결 확인 방법: 터미널에서 해당 폴더로 이동 후 아래 입력
```
cd "C:\Users\[사용자명]\donghoi-on-chatbot"
git remote -v
```
`seoulclub-on/26seoulclub` 주소가 보이면 정상입니다.

보이지 않으면 아래 명령으로 연결:
```
git remote add origin https://github.com/seoulclub-on/26seoulclub.git
```

### 1-4. Claude Code 설치
1. Windows 터미널(PowerShell)에서 아래 입력:
   ```
   npm install -g @anthropic/claude-code
   ```
2. 설치 완료 확인:
   ```
   claude -v
   ```
   버전 번호가 나오면 설치 완료

### 1-5. Claude Code 로그인 (최초 1회)
Claude Code를 쓰려면 Anthropic 계정이 필요합니다.

**방법 A — claude.ai 계정 이용 (권장)**
- [claude.ai](https://claude.ai) 에서 **Pro** 또는 **Max** 플랜 구독 후
- 터미널에서 `claude` 입력 → 로그인 화면에서 계정 연결

**방법 B — Anthropic API 키 이용**
- [console.anthropic.com](https://console.anthropic.com) → API Keys → API 키 발급
- 터미널에서:
  ```
  set ANTHROPIC_API_KEY=발급받은_키
  ```

## 2. 프로젝트 파일 구조

코드 수정을 요청하기 전, 어떤 파일이 어떤 역할인지 파악해두면 Claude Code에게 더 정확하게 요청할 수 있습니다.

```
donghoi-on-chatbot/
│
├── config.json              ← 챗봇 설정 (노션 URL, 구글 캘린더, AI 모델 등)
├── forbidden_words.json     ← 초기 시드 파일 (운영 중 관리는 대시보드 → 금지어 사전)
├── situation_keywords.json  ← 초기 시드 파일 (운영 중 관리는 대시보드 → 분류 키워드)
├── vercel.json              ← Vercel 배포 설정
├── requirements.txt         ← Python 패키지 목록
├── .env.example             ← 환경변수 목록 예시 (실제 값은 Vercel 대시보드에서 관리)
│
├── api/                     ← API 서버 (Vercel이 실행하는 핵심 코드)
│   ├── chat.py              ← 챗봇 대화 처리
│   ├── admin.py             ← 관리자 대시보드 API
│   ├── sync_notion.py       ← 수동 노션 동기화 ("지금 갱신" 버튼)
│   └── cron/
│       └── sync_notion.py   ← 자동 동기화 (매일 새벽 4시 크론)
│
├── collectors/              ← 데이터 수집기
│   ├── notion_collector.py  ← 노션 페이지 수집
│   ├── calendar_collector.py← 구글 캘린더 수집
│   ├── google_sheet_collector.py ← 구글 스프레드시트 수집
│   ├── pdf_collector.py     ← PDF 파일 수집
│   ├── docx_collector.py    ← Word 파일 수집
│   ├── excel_collector.py   ← Excel 파일 수집
│   └── airtable_collector.py← Airtable 수집 (현재 비활성)
│
├── storage/                 ← Supabase DB 연결 및 쿼리
├── public/                  ← 화면 파일
│   ├── widget.html          ← 챗봇 위젯 (노션에 임베드되는 화면)
│   └── admin/              ← 관리자 대시보드 화면
│       ├── index.html
│       ├── login.html
│       ├── app.js
│       └── styles.css
│
├── chatbot_engine.py        ← 챗봇 핵심 로직 (검색 → 프롬프트 → Claude API 호출)
├── auth.py                  ← 관리자 인증
├── build_embeddings.py      ← 문서 임베딩(벡터화) 처리
├── hybrid_search.py         ← 검색 엔진 (벡터+키워드 혼합)
├── prompt_builder.py        ← Claude API에 보내는 프롬프트 구성
├── failure_analyzer.py      ← 실패 원인 분류 로직
└── models/, utils/          ← 내부 공통 코드
```

### 파일 수정 기준

| 구분 | 파일 | 방법 |
|------|------|------|
| **직접 수정 가능** | `config.json` | GitHub 웹 편집기 또는 텍스트 편집기로 직접 수정 |
| **Claude Code로 수정** | `api/`, `collectors/`, `public/`, `*.py` 전체 | 터미널에서 Claude Code 실행 후 요청 |
| **수정하지 않음** | `forbidden_words.json`, `situation_keywords.json`, `vercel.json`, `requirements.txt`, `.env.example` | 앞 두 파일은 최초 DB 시드용이라 수정해도 런타임에 반영되지 않음(대시보드에서 관리). 나머지는 잘못 수정 시 배포가 깨질 수 있음 |

## 3. Windows 터미널에서 Claude Code 시작하기

### 매번 하는 절차

1. **Windows 터미널(PowerShell)** 또는 **명령 프롬프트** 열기
   - 시작 메뉴에서 "PowerShell" 검색 후 실행

2. **프로젝트 폴더로 이동**
   ```
   cd "C:\Users\[사용자명]\donghoi-on-chatbot"
   ```
   (폴더 경로는 실제 저장 위치에 맞게 수정)

3. **Claude Code 시작**
   ```
   claude
   ```

4. **대화 시작** — 프롬프트(`>`)가 나타나면 한국어로 요청하면 됩니다.
   ```
   > 챗봇 답변이 너무 길어. 최대 300자로 줄여줘
   ```

5. **종료**: `/exit` 입력 또는 `Ctrl + C`

### 유용한 명령어

| 명령어 | 설명 |
|--------|------|
| `claude` | Claude Code 시작 |
| `/help` | 도움말 |
| `/clear` | 대화 내용 초기화 (새 주제 시작 시) |
| `/exit` | 종료 |

## 4. Claude Code로 작업하기

### 4-1. 요청하는 방법

Claude Code는 프로젝트 전체 코드를 읽고 문맥을 파악합니다. 파일 이름을 몰라도 됩니다. 어떤 상황인지, 어떻게 바뀌면 좋겠는지를 자연어로 설명하면 됩니다.

**기본 요청 형식:**
```
현재 [어떤 상황]인데, [어떻게 바꾸고 싶다].
```

### 4-2. 작업 유형별 요청 예시

**버그 수정**
```
> 챗봇 위젯에서 질문을 보내면 "오류가 발생했습니다"라고 나와. 원인을 찾아서 고쳐줘.
```
```
> 관리자 대시보드에서 "지금 갱신" 버튼을 눌러도 아무 반응이 없어. 고쳐줘.
```

**기능 수정**
```
> 챗봇 답변 끝에 항상 "문의: 02-1234-5678" 문구를 추가하고 싶어.
```
```
> 현재 사용자가 같은 질문을 2번 반복하면 경고하는데, 3번으로 늘려줘.
```

**기능 추가**
```
> 관리자 대시보드에 "오늘 질의 건수"를 보여주는 간단한 숫자 카드를 홈 화면에 추가해줘.
```
```
> 챗봇이 답변할 수 없을 때 "죄송합니다" 대신 "확인 후 안내드리겠습니다"로 문구를 바꾸고 싶어.
```

**내용 확인**
```
> 챗봇이 임베딩 실패했을 때 어떻게 처리하는지 코드 흐름을 설명해줘.
```
```
> config.json에서 similarity_threshold가 어떤 역할인지 알려줘. 지금 값 0.55가 적절한지도 판단해줘.
```

### 4-3. Claude Code 활용 팁

- **구체적일수록 좋습니다**: "챗봇이 이상해" 보다 "챗봇이 노션에 있는 내용을 못 찾는 것 같아. 동기화는 정상인데 검색이 안 되는 것 같아"가 더 정확한 수정을 이끌어냅니다.
- **수정 내용을 확인하세요**: Claude Code가 수정을 완료하면 어떤 파일을 어떻게 바꿨는지 설명합니다. 이상하다 싶으면 `/clear` 후 다시 요청하거나, 원래대로 되돌려달라고 요청하세요.
- **한 번에 하나씩**: 여러 수정을 동시에 요청하면 결과 확인이 어렵습니다. 하나 수정 → 배포 확인 → 다음 수정 순서로 진행하세요.

## 5. 수정 후 배포하기

코드를 수정했으면 GitHub에 올려야 Vercel이 자동으로 새 버전을 배포합니다.

### 5-1. Claude Code 안에서 바로 커밋·푸시

Claude Code가 파일을 수정한 직후, 아래처럼 요청하면 됩니다:
```
> 방금 수정한 내용을 GitHub에 올려줘. 커밋 메시지는 "챗봇 답변 길이 제한 수정"으로.
```
Claude Code가 `git add` → `git commit` → `git push`까지 자동으로 처리합니다.

### 5-2. 터미널에서 직접 하는 방법

직접 하고 싶을 때는 아래 명령을 순서대로 입력합니다:
```
git add .
git commit -m "수정 내용 한 줄 요약"
git push
```

### 5-3. 배포 확인

1. [vercel.com](https://vercel.com) → 해당 프로젝트 → **Deployments** 탭
2. 목록 맨 위 항목이 **"Ready"**(초록 체크)이면 배포 성공
3. **"Error"**(빨간 ×)이면 해당 항목 클릭 → 빌드 로그 확인
   - 오류 메시지를 복사해서 Claude Code에게 붙여넣고 "이 오류 고쳐줘" 요청

### 5-4. 배포 완료 후 동작 확인

`{배포주소}/widget.html` 에서 직접 질문 1개 보내서 정상 응답 확인

## 6. 트러블슈팅

### a) 챗봇이 응답하지 않을 때

아래 순서로 원인을 확인합니다:

**① 위젯 오류 메시지 확인**
- "오류가 발생했습니다" → 서버 문제 (②로 이동)
- 아무 반응 없음 → 네트워크 또는 Vercel 문제 (②로 이동)
- 답변은 오는데 내용이 이상함 → 지식DB 문제 (사용 매뉴얼 4-2 "지금 갱신" 참고)

**② Vercel 배포 상태 확인**
- [vercel.com](https://vercel.com) → 프로젝트 → **Deployments** 탭
- 최신 배포가 "Error" → 빌드 로그 복사 → Claude Code에게 "이 오류 고쳐줘" 요청
- 최신 배포가 "Ready" → ③으로 이동

**③ Supabase 상태 확인**
- [supabase.com](https://supabase.com) → 해당 프로젝트 확인
- "Project paused" 메시지 → **6-c 항목** 참고하여 복구
- 정상 → ④로 이동

**④ Anthropic API 크레딧 확인**
- [console.anthropic.com](https://console.anthropic.com) → **Billing** 탭
- 크레딧이 0이거나 한도 초과 → 크레딧 충전

**⑤ 위 모두 정상인데 안 될 때**
```
> 챗봇 위젯에서 질문하면 [오류 메시지 내용]이 나와. 원인을 찾아서 고쳐줘.
```
Claude Code에게 오류 메시지를 그대로 붙여넣고 진단 요청

---

### b) 자동 동기화(크론)가 안 될 때

**① 어드민 대시보드 크론 배너 확인**
- 로그인 후 사이드바의 크론 상태 배너 확인
- 노란색/빨간색 → 마지막 실행 시각과 상태 확인

**② 수동으로 동기화 실행**
- 대시보드 → Knowledge Base → **"지금 갱신"** 클릭
- 오류 메시지가 나오면 → ③으로 이동
- 정상 완료 → 크론만 일시 오류였던 것 (Vercel이 가끔 크론을 건너뜀, 정상 범주)

**③ Vercel 크론 로그 확인**
- [vercel.com](https://vercel.com) → 프로젝트 → **Cron Jobs** 탭
- 실패한 실행 항목 클릭 → 로그 내용 복사 → Claude Code에 붙여넣어 진단 요청

> 무료(Hobby) 플랜은 Vercel 런타임 로그가 **1시간**만 보존됩니다. 크론 이상을 새벽 4시 이후 곧바로 확인하기 어렵습니다. 이상이 반복되면 Claude Code에게 크론 로직 자체를 진단 요청하세요.

### c) Supabase 프로젝트 일시정지 복구

DB 용량(500 MB)이 초과되면 Supabase가 프로젝트를 자동 일시정지합니다. 이 경우 챗봇과 대시보드가 모두 중단됩니다.

**복구 절차:**

1. [supabase.com](https://supabase.com) 로그인 → 해당 프로젝트 선택
2. 화면에 **"Project paused"** 또는 **"Restore project"** 버튼이 표시됨
3. **"Restore project"** 버튼 클릭
4. 복구 완료까지 **5~10분** 대기 (자동으로 진행)
5. 완료 후 챗봇 위젯에서 질문 1개로 정상 동작 확인

**복구 후 반드시 할 것: 용량 확보**
복구해도 용량이 여전히 초과 상태면 곧 다시 일시정지됩니다. 복구 직후 아래 **6-d 항목**을 따라 용량을 정리하세요.

### d) DB 용량 임박 시 정리

Supabase 무료 플랜 한도는 **500 MB**입니다.
사용량이 75%(375 MB) 이상이면 정리를 시작하는 것이 좋습니다.

**현재 용량 확인:**
[supabase.com](https://supabase.com) → 프로젝트 → **Settings → Usage**

**정리 방법 ① — 어드민 대시보드에서 삭제 (권장)**
- 대시보드 → **KB 데이터목록** → 더 이상 사용하지 않는 파일·구글시트 문서 삭제
- 노션 소스 문서는 노션에서 직접 페이지를 삭제한 뒤 "지금 갱신"을 눌러 DB에서도 제거

**정리 방법 ② — Supabase SQL Editor에서 직접 정리**

대시보드로 삭제할 수 없는 항목(크론 기록 등)은 Supabase SQL Editor를 사용합니다:

1. [supabase.com](https://supabase.com) → 프로젝트 → 좌측 메뉴 **SQL Editor**
2. 아래 쿼리 중 필요한 것을 붙여넣고 **Run** 클릭

```sql
-- 크론 실행 기록 중 90일 초과 항목 삭제 (안전)
DELETE FROM cron_run_log
WHERE created_at < NOW() - INTERVAL '90 days';
```

```sql
-- 현재 테이블별 용량 확인 (삭제 전 현황 파악용)
SELECT
  relname AS table_name,
  pg_size_pretty(pg_total_relation_size(relid)) AS total_size
FROM pg_catalog.pg_statio_user_tables
ORDER BY pg_total_relation_size(relid) DESC;
```

> **주의**: SQL Editor에서 실행한 쿼리는 즉시 반영되며 되돌릴 수 없습니다. 확신이 없으면 Claude Code에게 "이 쿼리가 안전한지 확인해줘" 라고 먼저 물어보세요.

**정리 방법 ③ — Claude Code에게 진단 요청**

어떤 데이터가 얼마나 차지하는지 파악하고 싶을 때:
```
> Supabase DB 용량이 많이 찼어. 어떤 테이블이 가장 많이 차지하는지 확인하는 SQL을 알려주고, 안전하게 정리할 수 있는 방법을 알려줘.
```

## 7. 자주 받는 질문

**Q. Claude Code가 코드를 잘못 수정했을 때 어떻게 되돌리나요?**

A. Claude Code에게 직접 요청합니다:
```
> 방금 수정한 내용이 마음에 안 들어. 원래대로 되돌려줘.
```
또는 GitHub에서 이전 커밋으로 되돌릴 수 있습니다:
```
git revert HEAD
git push
```
Claude Code에게 "직전 커밋으로 되돌려줘"라고 요청해도 됩니다.

**Q. 수정 내용을 배포하기 전에 미리 테스트할 수 있나요?**

A. GitHub에 푸시하면 Vercel이 자동으로 **프리뷰 배포**(별도 임시 URL)를 생성합니다. 운영 중인 챗봇에는 영향이 없으므로, Vercel 대시보드의 Deployments 목록에서 프리뷰 URL로 먼저 확인한 뒤 이상 없으면 사용하면 됩니다.

**Q. config.json을 수정했는데 적용이 안 돼요.**

A. `config.json`은 Vercel 재배포가 있어야 반영됩니다. GitHub에서 파일을 수정·커밋하면 자동으로 재배포됩니다. (대시보드의 "지금 갱신"만으로는 `config.json` 변경이 반영되지 않습니다.)

**Q. Anthropic API 크레딧은 어디서 충전하나요?**

A. [console.anthropic.com](https://console.anthropic.com) → **Billing** → **Add credits**에서 충전합니다.
