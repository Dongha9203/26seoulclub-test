# 동아리ON 챗봇 — 운영자 사용 매뉴얼

> 관리자 대시보드와 챗봇 위젯을 운영자가 직접 사용하는 방법을 안내합니다.
> (배포/환경설정 등 1회성 인프라 작업은 별도 인수인계 매뉴얼을 참고하세요.)

---

## 0. 화면 구성과 주소(URL)

배포된 주소를 `{배포주소}`라고 하면(예: `https://26seoulclub.vercel.app`):

| 화면 | 주소 | 사용자 |
|---|---|---|
| 챗봇 위젯 | `{배포주소}/widget.html` | 동아리원 (노션 페이지에 임베드되어 보임) |
| 관리자 로그인 | `{배포주소}/admin/login.html` | 운영자 |
| 관리자 대시보드 | `{배포주소}/admin/index.html` (로그인하면 자동 이동) | 운영자 |

**주의**: 로그인 세션은 브라우저 탭에만 저장됩니다(12시간 후 자동 만료). 브라우저
탭/창을 닫으면 세션이 사라지므로 다시 열면 로그인을 다시 해야 합니다.

---

## 1. 로그인 / 로그아웃 / 비밀번호 변경

1. `{배포주소}/admin/login.html`에서 이메일·비밀번호로 로그인
2. 좌측 메뉴 맨 아래 **비밀번호 변경**에서 현재 비밀번호 확인 후 새 비밀번호(8자
   이상)로 변경 가능
3. **로그아웃** 버튼으로 즉시 세션 종료, 또는 12시간이 지나면 자동 만료되어
   다음 동작 시 로그인 화면으로 돌아갑니다

## 2. 모니터링

### 2-1. 일별 질의/응답 건수
- 날짜별로 챗봇에 들어온 질문 수를 보여줍니다. 운영 추이 파악용입니다.
- 기간별 조회조건: 최대 **3개월**까지 조회 가능. 비워두면 전체 기간(최근 30건
  단위 페이지네이션) 조회.
- "더보기"/"이전"/"처음 페이지로 이동"으로 페이지 이동 (1페이지 = 최대 30건)

### 2-2. 질의-답변 연계조회
- 사용자 질문과 챗봇이 실제로 보낸 답변을 한 줄에 짝지어 보여줍니다.
- 기간별 조회조건: 최대 **1개월**까지 (다른 화면보다 좁음 — 데이터 양이 많아서
  그렇게 설계되어 있습니다)

> 모니터링 두 화면 모두, 자료는 **최대 1년까지 보관되며 그 이후 자동 삭제**됩니다.
> 1년이 지난 기록은 집계에 나타나지 않습니다.

## 3. 조치관리

### 3-1. 불완전 답변 조회
검색에 실패했거나, 질문이 너무 모호해서(예: "이거 뭐예요?"처럼 무엇을 묻는지 알
단어가 없는 경우) 챗봇이 직접 답하지 않고 운영팀 연락처만 안내한 건들입니다.

### 3-2. 미해결 답변 조회
지식 베이스(Knowledge Base)에 관련 내용이 없거나, 프로그램과 무관한 질문(예:
"오늘 날씨 어때요?")이라 챗봇이 답할 수 없어서 운영팀 연락처만 안내한 건들입니다.

이 두 화면 모두 기간별 조회조건은 최대 **3개월**입니다. 목록의 **"삭제"** 버튼은
노션 문서를 보강하거나 필요한 조치를 끝낸 뒤 "이 항목은 처리 완료했다"고 체크하는
용도입니다 — 누른다고 실제 질의응답 기록이나 통계(일별 건수, 원인별 집계 리포트)가
사라지는 게 아니라, **이 목록에서만** 빠지고 사라집니다.

### 3-3. 원인별 집계 리포트
검색 실패 원인을 5가지로 분류해 건수를 보여줍니다 (최대 **12개월** 조회 가능,
다른 화면보다 넓게 설계됨).

| 원인 | 의미 | 대응 |
|---|---|---|
| 지식DB공백 | 동아리ON 운영 관련 질문인데 지식 베이스에 그 내용을 다루는 문서가 아예 없음 | 해당 내용을 지식 베이스에 새로 등록하면 다음부터 답변 가능 |
| 검색실패 | 관련 문서는 있지만 그 안에서 구체적인 답을 찾지 못함 | 기존 문서 내용을 더 구체적으로 보강 |
| 질문모호성 | 질문에 핵심 단어가 거의 없어 무엇을 묻는지 알 수 없음 | 사용자가 질문을 구체적으로 다시 입력하면 해결되는 경우가 많음 — 별도 조치 불필요 |
| 정책밖요청 | 날씨, 요리 등 동아리ON과 전혀 무관한 질문 | 정상 동작, 조치 불필요 |
| API오류 | Claude API 호출이 일시적으로 실패(서버 과부하/네트워크 등) | 한두 건은 정상. 짧은 시간에 많이 쌓이면 API 사용량·네트워크 상태 확인 필요 |

## 4. 운영설정

### 4-1. 담당자 연락처
챗봇이 답변하지 못하거나 운영팀 연결이 필요할 때 사용자에게 보여주는 연락처
(이름/주소/전화번호/이메일 여러 개/운영시간)입니다. 저장하면 **다음 질문부터
바로** 반영됩니다.

### 4-2. Knowledge Base (챗봇이 답변의 근거로 쓰는 문서 관리)

- **지금 갱신**: 노션 페이지를 수정한 직후, 매일 자동 갱신(새벽 4시)을 기다리지
  않고 바로 반영하고 싶을 때 누릅니다.
- **파일 업로드**: `.docx` / `.pdf` / `.xlsx` 파일을 업로드하면 자동으로 내용이
  저장됩니다. **HWP/HWPX는 직접 업로드할 수 없어서**, PDF나 텍스트 파일로 미리
  변환한 뒤 업로드해야 합니다.
- **구글 스프레드시트**: "누구나 보기" 공개 공유 링크 URL을 입력하면 가져옵니다
  (비공개 시트는 지원되지 않습니다). 시트의 행(row) 1개 = 문서 1개로 등록됩니다.
- **같은 이름으로 다시 업로드/같은 시트 URL로 다시 가져오기** 하면 기존 내용을
  덮어쓰고 새 내용으로 교체됩니다(중복 등록되지 않음).
- 소제목 구조가 없는 800자 이상의 긴 문서는 "(파트 1)", "(파트 2)"처럼 검색
  정확도를 위해 자동으로 여러 건으로 나뉘어 등록됩니다 — 정상 동작입니다.

⚠️ **중요**: 파일/구글시트로 업로드한 문서는 목록에는 바로 나타나지만, 그 옆의
**"갱신"** 버튼(또는 한 번에 처리하는 **"전체 갱신"** 버튼)을 눌러야 챗봇이 실제로
그 내용을 찾아 답변할 수 있습니다. 누르지 않으면 목록에는 보여도 챗봇은 그 내용을
모릅니다. 시트처럼 한 번에 문서가 많이 생기면 "전체 갱신"을 사용하세요.

- 목록에서 노션 소스 문서는 조회만 가능하고 "삭제" 버튼이 꺼져 있습니다(노션에서
  직접 수정해야 함). 파일/구글시트로 등록한 문서는 "삭제"로 바로 제거할 수
  있습니다.

### 4-3. 톤 설정 (브랜드 톤 8요소)
챗봇이 모든 답변에서 항상 지키는 기본 말투·태도 규칙입니다(예: 존댓말만 쓴다,
추측하지 않고 문서 내용만 답한다 등). 8개 항목(성격/언어순도/VIP 일관성/격식/
채널/감정노동/역할/사실성)을 각각 글로 수정할 수 있고, 저장하면 다음 질문부터
바로 적용됩니다.

### 4-4. 분류 키워드 관리
- **상황 분류 키워드** (부분일치): 정책위반요청(예: 대리 출석 요청), 상담원
  연결 요청, 감사 인사, 단순 거절 — 질문 문장에 등록한 키워드가 일부라도 포함되면
  그 상황으로 분류됩니다(예: "대신 출석"을 등록하면 "친구가 대신 출석 체크해줘도
  되나요?"에도 매칭). 정책위반요청으로 분류되면 챗봇이 명확히 거절하는 톤으로
  답합니다.
- **금지어 사전** (부분일치): 욕설/혐오표현/협박. 매칭되면 Claude API를 호출하지
  않고 곧바로 운영팀 안내 응답으로 처리됩니다(비용 절감 목적의 1차 방어선).
- 두 곳 모두 textarea에 **한 줄에 키워드 하나씩** 입력하고 저장하면 다음 질문부터
  즉시 반영됩니다.

### 4-5. 챗봇 운영지침 (남용 방지 설정)
- **최대 질문 글자수** (1~2000자): 위젯으로 들어오는 질문의 글자수 제한
- **분당 요청 제한** (1~100회): 같은 사용자(세션) 기준으로 1분에 몇 번까지
  질문할 수 있는지
- 저장하면 재배포 없이 다음 질문부터 바로 적용됩니다

## 5. 챗봇 위젯 (동아리원이 보는 화면)

- 노션 페이지에 임베드된 입력창에 질문을 입력하고 Enter(전송) 또는 전송 버튼을
  누르면 답변이 옵니다. Shift+Enter는 줄바꿈입니다.
- 답변은 한 글자씩 실시간으로 이어서 표시됩니다(스트리밍).
- 같은 브라우저에 세션 ID가 저장되어, 같은 사람이 반복해서 같은 질문을 하면
  챗봇이 이를 인식합니다.
- 답변 안에 관련 노션 문서로 바로 이동하는 링크가 포함될 수 있습니다.

## 6. 사용 시 주의사항 모음

- **로그 보관 기간**: 모든 질의응답 기록은 1년 보관 후 자동 삭제됩니다. 통계를
  장기간 보고 싶으면 1년이 지나기 전에 따로 백업해두세요.
- **자동 갱신 주기**: 노션 동기화는 하루 1번(새벽 4시)만 자동으로 일어납니다.
  급하게 반영해야 하면 Knowledge Base 화면의 "지금 갱신"을 사용하세요.
- **노션 페이지 공유**: 노션에 새 페이지를 만들거나 다른 곳에서 옮겨온 페이지를
  챗봇이 참고하게 하려면, 그 페이지의 "..." → "연결(Connections)"에 챗봇
  통합(integration)이 추가되어 있어야 합니다. 빠지면 그 페이지 내용은 영원히
  검색되지 않습니다.
- **Airtable은 못 읽지만, 노션 자체 데이터베이스(자료실)는 읽습니다**: 노션
  페이지에 Airtable 표/폼을 임베드한 경우 표 안의 실제 내용은 검색되지 않고
  링크만 노출됩니다(이건 여전히 못 읽습니다). 반면 노션 자체의 데이터베이스
  (표 보기) 기능으로 만든 자료(예: "자료실")는 각 행(row)을 별도 페이지로 보고
  그 안의 내용을 챗봇이 읽어 검색에 활용하며, 행 추가/삭제/수정도 매일 새벽
  자동 갱신에 반영됩니다. 급하게 반영해야 하면 "지금 갱신"을 누르세요.
- **업로드 후 "갱신" 누르기**: 파일/구글시트 업로드는 "갱신"(또는 "전체 갱신")을
  눌러야 챗봇 검색에 실제로 반영됩니다 (4-2 참고).
- **조치관리 화면의 "삭제"**: 통계나 원본 기록을 지우는 게 아니라, 처리 완료
  표시로 목록에서만 빼는 기능입니다.
- **로그인 정보 보안**: 관리자 비밀번호는 외부에 공유하지 말고, 의심되는 로그인
  시도가 보이면 즉시 비밀번호를 변경하세요.
